## Context

V1 提示词（`optimize-report-prompt-for-lower-models`）为低级模型提供了 Markdown 骨架和字数约束，解决了"结构稳定性"问题。但用户实际阅读后反馈：

1. 建议通用化——换一组优势也能套用，缺乏"这是专门写给我"的质感
2. 优势组合的"化学反应"描述不深——只讲"思考+执行"这类大类搭配
3. "领导力"维度缺失——用户期望看到自己属于哪种领导风格
4. 语气偏"教练训人"——缺乏共情和邀请感

V2 的核心思路：**给 AI 更多结构化上下文 + 用心理咨询式语气框架**。

## Goals / Non-Goals

**Goals:**
- 让 AI 能基于优势定义 + 领域本质做"有知识基础的分析"，而非词汇猜测
- 让报告显式回答"我属于哪种领导力风格"
- 让语气从"诊断"转向"看见与邀请"，降低读者的防御
- 对低级模型（GLM-4-Flash、GPT-3.5）依然有效

**Non-Goals:**
- 不做模型路由（不根据模型名切换 prompt 版本）
- 不新增 API 调用轮次
- 不引入外部 prompt 模板库
- 不改造 strengths.json 数据结构（只读取现有 description 字段）

## Decisions

### Decision 1: 从 strengths.json 读取官方优势定义

**选择**: 直接 `import` 中英文 strengths.json，读取每个主题的 `description` 字段注入到 user message。

**理由**: 数据已存在于项目中，无需重复维护。每个优势一句话定义（约 12 字）对 token 成本影响小（5 个优势 +60 字），但显著降低低级模型对"思维/个别/审慎"这类专业术语的误读率。

**替代方案**: 在 system message 里提供 34 个优势的完整词典——token 消耗过大，且用户只关心前 5 项。

### Decision 2: 硬编码领域档案而非读 i18n

**选择**: 在 `aiService.ts` 里用 `DOMAIN_PROFILES_ZH/EN` 常量存储"本质 + 领导力风格"，不放 i18n。

**理由**: 这是提示词配置数据，不是 UI 文案。放 i18n 会让译者误以为可以改措辞，实际上这里的文字是经过设计、用于提示 AI 的，改动需要整体重新评估效果。常量放在服务层更清晰。

**替代方案**: 放到 strengths.json 里新增 `domains` 字段——会让数据文件承担混合职责。

### Decision 3: 澄清"领导力不是第五领域"

**选择**: 在 system message 里显式说明"领导力不是独立领域，而是每个领域在领导者身上的展开方式"。

**理由**: 低级模型面对"4 个领域 + 领导力风格"这种结构时，可能虚构一个"领导力领域"。显式否定能防止幻觉。

### Decision 4: 主导领域 = 最大计数的领域集合

**选择**: 并列时全部列出（如"战略思维 + 执行力"），让 AI 产出"混合型领导风格"描述。

**理由**: 盖洛普的 4 种领导风格框架承认混合型。前 5 大优势按 2-2-1-0 分布时常出现并列主导，强制选一个会丢失重要信息。

### Decision 5: 章节重命名 + 新增开场白/结语

**选择**: "盲点" → "过度工作的保护机制"；"建议" → "你已经拥有的力量与适合的战场"；加开场 + 结语。

**理由**: 参考了心理咨询报告的写作原则——用"被误用的保护机制"替代"缺陷"；用"已有资源"替代"建议"；让报告以温暖的确认收尾。这种重命名对低级模型的遵从度没有负面影响（仍然是明确的 `##` 骨架）。

### Decision 6: 报告中强制显式报出领导力风格

**选择**: 在 "## 我看见的你" 章节要求用一句话明确指出"基于主导领域，你最接近哪种盖洛普领导力风格"。

**理由**: 这是本次升级最直接的用户可见价值。不强制说出来，低级模型常常隐含带过而不点明。

## Risks / Trade-offs

- **提示词 token 增加** → V1 约 800-1000 tokens，V2 约 1200-1500 tokens，增加约 400-500 tokens。GLM-4-Flash 免费方案下这个增量可接受，但注意 context window 不能无限扩。→ 缓解：保持领域档案精简（每个 2 句），不展开优势的 `longDescription`。
- **低级模型的遵从能力可能见顶** → 章节数从 4 增至 6（开场 + 我看见的你 + 保护机制 + 力量与战场 + 本周一步 + 结语），低级模型可能漏写开场或结语。→ 缓解：明确字数范围 + 示例占位符提示 + 骨架在 system message 中完整呈现。
- **语气"邀请式"可能让报告感觉软** → 读者想要的是"诚实的看见"，而不是"温和的安慰"。→ 缓解：规则中明确"不要和稀泥"，保留"可能消耗你的方向"这类坦诚判断，只是用邀请式措辞包装。
